2. Features Implemented

A. Authentication & Security (JWT)

  • Enforced Customer Registrations: The public /register endpoint restricts new accounts strictly to the customer role, completely ignoring role-selection tampering in payloads.
  • Seeded Administrator Model: Administrator accounts can only be generated through database seeds or direct database additions, securing the route from illegal escalations.
  • Role-aware Route Guards: Client routes redirect unauthorized guests to appropriate contexts, while backend API middleware enforces JWT validation (protect) and administrator clearance checks (adminOnly).

B. Advanced Reservation Availability Check

  • Double-Booking Prevention Check: The API runs a pre-check query to verify if the selected table is occupied for the specified date and time block.
  • Database Compound Constraints: A compound unique index has been defined on the Mongoose Reservation Schema ({table, date, timeSlot}) which triggers a MongoDB duplicate key collision failure to handle parallel processing race conditions safely.
  • Table Capacity Validation: Guests limits are checked against the selected table's absolute capacity prior to slot reservation.

C. Dynamic Administrator Dashboard Panel

  • Full View & Filter: Displays all active table reservations across all customers with date-specific filter capability.
  • Reservation Cancellation: Allows the admin to cancel any reservation.
  • Edit & Reassign Panel: Implements a visual glassmorphic edit modal popup allowing administrators to change the assigned table, time slot, guest counts, and dates. All safety checks run upon modification to prevent collisions.

3. Technical Fixes & Code Improvements Audited

A comprehensive audit pass was performed to clean the codebase and ensure high production readiness:

  • Input Validation: Integrated checks to validate date format (YYYY-MM-DD), check valid time slot limits, and verify positive integer values (>0) for guests, throwing detailed 400 Bad Request messages on failure.
  • Falsy Guest Bug Resolution: Replaced simple truthiness updates (if (guests)) with explicit !== undefined checks to allow users to scale guest counts to valid numbers without updates silently skipping.
  • Centralized Error Handling: Refactored route catch-blocks to use next(error) instead of dead generic internal Server Error handlers, allowing centralized Express middleware to intercept and log unexpected crashes.
  • Single-Page Routing Fix: Configured a Vercel routing rule (vercel.json) directing client requests to /index.html, resolving 404 page-refresh failures.
